# 객체 지향 프로그래밍 Object-oriented programming (OOP)

## 💡 객체 지향 프로그래밍이란?
*프로그램을 명령어 목록이 아닌, 객체끼리 상호작용으로 데이터를 처리하는 프로그래밍 방식
==========================================================
### 주요 개념
객체:데이터와 그 데이터를 처리하는 기능을 묶은 단위

핵심4가지 특성:
1. 캡슐화: 관련 데이터를 하나로 묶고 내부 동작을 숨겨줌.
2. 상속: 상위 클래스의 데이터의 특성을 하위 클래스 그대로 받아서 코드 재사용 및 확장을 쉽게함.
3. 다향성: 같은 이름의 메서드(객체의 동작이나 기능) 상황에 따라서 다르게 동작하도록 유연성을 높혀줌.
4. 추상화: 복잡한 구조는 숨기고 필요한 내용만 보여줘서 모델링을 단순화함

#### 장단점

*OOP의 장점:
코드 재사용이 용이함, 유지보수 및 확장이 쉬움,대형 프로젝트에 적합함.


*OOP의 단점
설계가 복잡함,메모리 사용량이 매우 높음,절차비향에 비해서 속도가 느림
